# DevSimulator 사용 가이드

> 프로그래밍 없이 PLC/장비 동작을 시뮬레이션하는 방법을 설명합니다.

---

## DevSimulator란?

실제 PLC나 장비가 없어도 **소프트웨어로 장비인 척** 해주는 프로그램입니다.

예를 들어, PLC와 통신하는 WPF 앱을 만들고 있는데 PLC가 없다면 — DevSimulator를 켜두면 WPF 앱은 실제 PLC와 통신하는 것처럼 동작합니다.

```
[WPF 앱] ──── 네트워크 ──── [DevSimulator] ← 실제 PLC 대신
                                ↕
                          블록코딩으로
                          직접 정의한 동작
```

---

## 1. 시작 전 준비

### 필요한 것
- Windows 10/11 PC
- [.NET 8 Desktop Runtime](https://dotnet.microsoft.com/ko-kr/download/dotnet/8.0) 설치
- Visual Studio 2022 (빌드용, 처음 한 번만)

### 처음 실행하기
1. GitHub에서 코드 다운로드 (초록색 `Code` 버튼 → `Download ZIP`)
2. 압축 해제 후 `SimulatorProject.sln` 파일을 Visual Studio로 열기
3. **Ctrl+F5** 로 빌드 & 실행

## 2. 화면 구성

```
┌──────────────────────────────────────────────────────┐
│  [▶ 시작]  [⏹ 정지]  IP: 127.0.0.1  Port: 5000      │  ← 툴바
├──────────┬────────────────────────────┬───────────────┤
│          │                            │ 디바이스 모니터│
│ 노드     │                            │───────────────│
│ 팔레트   │   캔버스 (여기에 노드 배치)  │ Key   │ Value │
│          │                            │ D100  │  0    │
│✏️ Set   │                            │ D101  │  0    │
│⏱️ Wait  │                            │ M0    │  0    │
│🔀 Cond  │                            │       │       │
│🔴 End   │                            │       │       │
└──────────┴────────────────────────────┴───────────────┘
```

| 영역 | 설명 |
|---|---|
| **툴바** | 시뮬레이터 시작/정지, IP/Port 설정 |
| **노드 팔레트** | 사용 가능한 블록 목록 |
| **캔버스** | 블록을 드래그해서 동작 흐름을 만드는 공간 |
| **디바이스 모니터** | 레지스터(D100 등) 값을 실시간으로 보여줌 |

---

## 3. 노드(블록) 종류

| 노드 | 색상 | 역할 |
|---|---|---|
| **✏️ Set Value** | 주황 | 레지스터에 값을 설정합니다 |
| **⏱️ Wait** | 보라 | 지정한 시간(ms)만큼 기다립니다 |
| **🔀 Condition** | 진보라 | 조건에 따라 흐름을 나눕니다 (YES/NO) |
| **🔴 End** | 빨강 | 흐름을 종료합니다 |

## 4. 기본 사용법 — 단계별 예시

### 예시: "D100에 100을 넣고 2초 후 0으로 리셋"

#### Step 1: 노드 배치
팔레트에서 캔버스로 드래그:

```
✏️ Set Value
    ↓
⏱️ Wait
    ↓
✏️ Set Value
    ↓
🔴 End
```

#### Step 2: 각 노드 설정
캔버스에서 노드를 클릭하면 오른쪽 속성 패널에서 값을 바꿀 수 있습니다.

| 노드 | 설정값 |
|---|---|
| 첫 번째 Set Value | DeviceKey: `D100`, Value: `100` |
| Wait | DelayMs: `2000` (2초) |
| 두 번째 Set Value | DeviceKey: `D100`, Value: `0` |

#### Step 3: 시작
툴바에서 **▶ 시작** 클릭

→ D100이 100이 되고, 2초 후 0으로 바뀌는 것을 오른쪽 **디바이스 모니터**에서 확인할 수 있습니다.

## 5. Condition(조건 분기) 사용법

"M0이 1이면 D200=100, 아니면 D200=0" 을 만드는 방법:

```
🔀 Condition (DeviceKey: M0, Operator: Equal, Value: 1)
    │
    ├── YES → ✏️ Set Value (D200 = 100) → 🔴 End
    │
    └── NO  → ✏️ Set Value (D200 = 0)  → 🔴 End
```

> **Condition 설정 항목:**
> - `DeviceKey`: 비교할 레지스터 이름 (예: `D100`, `M0`)
> - `Operator`: 비교 방식
>   - `Equal` — 같으면
>   - `NotEqual` — 다르면
>   - `GreaterThan` — 크면
>   - `LessThan` — 작으면
> - `CompareValue`: 비교 기준값

## 6. 레지스터(디바이스) 이름 규칙

미쯔비시 PLC 방식을 따릅니다:

| 종류 | 이름 예시 | 용도 |
|---|---|---|
| **D** | D0, D100, D200 | 데이터 레지스터 (숫자 저장) |
| **M** | M0, M1, M100 | 내부 릴레이 (ON/OFF 상태) |
| **Y** | Y0, Y10 | 출력 |
| **X** | X0, X10 | 입력 |

> 이름은 알파벳 + 숫자로 자유롭게 쓸 수 있습니다. `D100`과 `D101`은 서로 다른 레지스터입니다.

## 7. IP/Port 설정

기본값은 `127.0.0.1:5000`입니다.

| 상황 | 설정 |
|---|---|
| 같은 PC에서 테스트 | IP: `127.0.0.1` (변경 불필요) |
| 다른 PC에서 접속 | IP: 이 PC의 LAN IP (예: `192.168.0.10`) |
| 포트 충돌 시 | Port를 `5001`, `5002` 등으로 변경 |

> 내 PC의 IP 확인: **윈도우 검색 → cmd → `ipconfig` 입력 → IPv4 주소 확인**

## 8. 시나리오 저장/불러오기

현재 버전에서는 `examples/scenario1.json` 파일 형식으로 저장됩니다.

**JSON 파일 구조 예시:**
```json
[
  { "$type": "SetValue", "DeviceKey": "D100", "Value": 100, ... },
  { "$type": "Wait", "DelayMs": 2000, ... },
  { "$type": "End", ... }
]
```

> 향후 업데이트에서 UI에서 저장/불러오기 버튼이 추가될 예정입니다.

## 9. 통신 테스트 (개발자용)

시뮬레이터가 실행 중일 때 Python으로 간단히 테스트할 수 있습니다:

```bash
cd examples
python test_slmp.py
```

**예상 결과:**
```
✅ 연결 성공

[테스트 1] D100 읽기 (초기값)
  D100 = 0

[테스트 2] D100 = 1234 쓰기
  쓰기 결과: ✅ 성공

[테스트 3] D100 읽기 (1234 확인)
  D100 = 1234  ✅ 정상
```

동시에 앱의 **디바이스 모니터**에서 D100 값이 1234로 바뀌는 것을 확인할 수 있습니다.

## 10. 자주 묻는 질문

**Q: 시작을 눌렀는데 연결이 안 돼요**
- Windows 방화벽이 차단했을 수 있습니다. `제어판 → Windows Defender 방화벽 → 앱 허용`에서 DevSimulator를 허용해 주세요.

**Q: Port 5000이 이미 사용 중이래요**
- 툴바에서 Port를 `5001`이나 `5002`로 바꾸고 다시 시작하세요.

**Q: 노드를 연결하려면 어떻게 하나요?**
- 현재 버전은 캔버스에 배치된 순서대로 자동 연결됩니다. 노드를 드래그해서 순서를 바꿀 수 있어요.

**Q: 실제 미쯔비시 PLC와 교체하려면?**
- 연결하는 WPF 앱에서 IP 주소만 PLC의 IP로 바꾸면 됩니다. 코드 수정이 필요 없어요.

**Q: 미쯔비시 외 다른 장비도 시뮬레이션 가능한가요?**
- 현재는 미쯔비시 SLMP 프로토콜만 지원합니다. 향후 Modbus TCP 등이 추가될 예정입니다.
